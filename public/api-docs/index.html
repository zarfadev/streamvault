StreamVault API

Programmatic video hosting. Upload, transcode, stream, and manage video infrastructure through a RESTful API with full HLS delivery and AI-powered features.

Quick Start Guide → Get API Key
v1.0 · REST · JSON · HTTPS
Overview

Introduction

The StreamVault API enables you to upload, manage, and stream video at scale. All communication is over HTTPS with JSON request/response bodies. Timestamps are UNIX epoch integers (seconds).

Base URL https://api.streamvault.link
Adaptive HLS
Auto-transcode to 360p-4K ABR with AES-128 encryption and signed token delivery.
AI Transcription
Whisper-powered speech-to-text in 11 languages. VTT subtitles via API.
Webhooks
HMAC-SHA256 signed event delivery for video.ready, transcription.complete and more.
All responses return Content-Type: application/json. Errors always include { "error": "human-readable message" }.
Security

Authentication

All authenticated endpoints require two headers: Authorization with your token and X-Workspace-Id with the target workspace.

Bearer Token (JWT)

HeaderAuthorization: Bearer eyJhbGciOiJIUzI1NiJ9...
WorkspaceX-Workspace-Id: ws_a1b2c3d4e5

Obtain via POST /auth/login. Access tokens expire in 15 min. Use POST /auth/refresh with your refresh token to obtain a new pair.

API Key Pro+

HeaderAuthorization: Bearer sv_live_a1b2c3d4e5f6g7h8...
WorkspaceX-Workspace-Id: ws_a1b2c3d4e5

API keys begin with sv_live_. The full key is shown once at creation. Keys are scoped to specific permissions (see below).

API keys are workspace-bound. The X-Workspace-Id header must match the workspace where the key was generated.
Security

API Key Scopes

API keys use granular scopes to restrict access. JWT tokens from login bypass scopes and have full workspace access.

ScopeAccess LevelEndpoints
videos:readRead video metadata, list, playback tokens, downloadGET /api/videos*, GET .../token
videos:writeUpdate metadata, tags, visibility, retry transcodePATCH /api/videos/:id, POST .../retry
videos:deleteDelete videos individually or in bulkDELETE /api/videos/:id, POST .../bulk
uploads:writeUpload new video files, import from URLPOST /api/upload, POST /api/import
analytics:readRead analytics reports, live viewers, CSV exportGET .../analytics, .../live, .../export.csv
Requests outside the key's scope return 403 with { "error": "Insufficient scope" }.
Getting Started

Quick Start Guide

Get from zero to your first API call in under 2 minutes.

Generate your API key

  1. Log in to your StreamVault account and go to Dashboard → Settings → API Keys
  2. Click "Create API Key", give it a name and select the scopes you need
  3. Copy the key immediately — it starts with sv_live_ and is shown only once
  4. Store it securely in environment variables (never commit to git)
  5. Use it in the Authorization: Bearer sv_live_... header with every request

First API call — List your videos

JavaScript
Python
curl -X GET BASE_URL/api/videos \
  -H "Authorization: Bearer sv_live_YOUR_API_KEY" \
  -H "X-Workspace-Id: YOUR_WORKSPACE_ID"

Import to Postman

We provide an OpenAPI 3.0 spec file you can import directly into Postman, Insomnia, or any API client.

  1. Download the spec: api-spec.yaml
  2. In Postman: Import → File → Select api-spec.yaml
  3. Set the baseUrl variable to https://api.streamvault.link
  4. Add your API key to the collection's Authorization tab as Bearer Token

Errors & Rate Limits

Standard HTTP status codes. All errors include a human-readable error field.

StatusMeaning
200Success
201Resource created
202Accepted — async job queued
204Deleted — no content
400Bad request — validation failed
401Unauthenticated — invalid or expired token
403Forbidden — insufficient plan, role, or scope
404Resource not found
409Conflict — duplicate resource
429Rate limited — check Retry-After header
500Internal server error
503Service unavailable — maintenance
Rate limits: 100 req/min (general), 10 req/min (upload), 5 req/min (auth). The X-RateLimit-Remaining and Retry-After headers are included in responses.
