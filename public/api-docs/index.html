StreamVault API

Hosting de video programático. Sube, transcodifica, transmite y gestiona tu infraestructura de video a través de una API RESTful con entrega HLS adaptativa y funciones potenciadas por IA.

Guía Rápida →
Visión General

Introducción

La API de StreamVault te permite subir, gestionar y transmitir video a escala. Toda la comunicación es sobre HTTPS con cuerpos de petición/respuesta en JSON. Los timestamps son enteros UNIX epoch (segundos).

URL Base
HLS Adaptativo
Auto-transcodificación a 360p-4K ABR con cifrado AES-128 y entrega con token firmado.
Transcripción IA
Speech-to-text con Whisper en 11 idiomas. Subtítulos VTT generados vía API.
Webhooks
Entrega de eventos firmados con HMAC-SHA256 para video.ready, transcription.complete y más.
Todas las respuestas devuelven Content-Type: application/json. Los errores siempre incluyen { "error": "mensaje legible" }.
Seguridad

Autenticación

Todos los endpoints autenticados requieren dos headers: Authorization con tu token y X-Workspace-Id con el workspace destino.

Bearer Token (JWT)

HeaderAuthorization: Bearer eyJhbGciOiJIUzI1NiJ9...
WorkspaceX-Workspace-Id: ws_a1b2c3d4e5

Obtenlo vía POST /auth/login. Los access tokens expiran en 15 min. Usa POST /auth/refresh con tu refresh token para obtener un nuevo par.

API Key Pro+

HeaderAuthorization: Bearer sv_live_a1b2c3d4e5f6g7h8...
WorkspaceX-Workspace-Id: ws_a1b2c3d4e5

Las API keys comienzan con sv_live_. La clave completa se muestra una sola vez al crearla. Las claves están limitadas por scopes específicos (ver abajo).

Las API keys están vinculadas a un workspace. El header X-Workspace-Id debe coincidir con el workspace donde se generó la clave.
Seguridad

Scopes de API Key

Las API keys usan scopes granulares para restringir acceso. Los tokens JWT del login omiten scopes y tienen acceso completo al workspace.

ScopeNivel de AccesoEndpoints
videos:readLeer metadata, listar, tokens de playback, descargaGET /api/videos*, GET .../token
videos:writeActualizar metadata, tags, visibilidad, reintentar transcodePATCH /api/videos/:id, POST .../retry
videos:deleteEliminar videos individual o masivamenteDELETE /api/videos/:id, POST .../bulk
uploads:writeSubir archivos, importar desde URLPOST /api/upload, POST /api/import
analytics:readLeer reportes, viewers en vivo, exportar CSVGET .../analytics, .../live, .../export.csv
folders:readListar carpetasGET /api/folders
folders:writeCrear, renombrar, eliminar carpetasPOST/PATCH/DELETE /api/folders
playlists:readListar y obtener playlistsGET /api/playlists*
playlists:writeCrear, editar, eliminar playlistsPOST/PATCH/DELETE /api/playlists
webhooks:readListar webhooks y entregasGET /api/webhooks*
webhooks:writeCrear, editar, eliminar webhooksPOST/PATCH/DELETE /api/webhooks
Peticiones fuera del scope de la clave devuelven 403 con { "error": "Insufficient scope" }.
Inicio

Guía Rápida

De cero a tu primera llamada API en menos de 2 minutos.

Genera tu API Key

  1. Inicia sesión en tu cuenta StreamVault y ve a Dashboard → Ajustes → API Keys
  2. Haz clic en "Crear API Key", dale un nombre y selecciona los scopes necesarios
  3. Copia la clave inmediatamente — empieza con sv_live_ y se muestra solo una vez
  4. Guárdala de forma segura en variables de entorno (nunca la subas a git)
  5. Úsala en el header Authorization: Bearer sv_live_... con cada petición

Primera llamada — Listar tus videos

JavaScript

curl -X GET BASE_URL/api/videos \
  -H "Authorization: Bearer sv_live_TU_API_KEY" \

print(f"Encontrados {len(data['videos'])} videos")

Importar a Postman

Proporcionamos un archivo de especificación OpenAPI 3.0 que puedes importar directamente en Postman, Insomnia o cualquier cliente API. También puedes descargar una colección Postman pre-armada.

  1. Descarga la spec: api-spec.yaml o usa el botón "Exportar a Postman" arriba
  2. En Postman: Importar → Archivo → Selecciona el archivo descargado
  3. Configura la variable baseUrl con la URL de este sitio
  4. Agrega tu API key en la pestaña de Authorization de la colección como Bearer Token

Referencia

Errores y Límites de Tasa

Códigos de estado HTTP estándar. Todos los errores incluyen un campo error legible.

EstadoSignificado
200Éxito
201Recurso creado
202Aceptado — trabajo async en cola
204Eliminado — sin contenido
400Petición inválida — validación fallida
401No autenticado — token inválido o expirado
403Prohibido — plan, rol o scope insuficiente
404Recurso no encontrado
409Conflicto — recurso duplicado
429Limitado — revisa header Retry-After
500Error interno del servidor
503Servicio no disponible — mantenimiento
Límites de tasa: 100 req/min (general), 10 req/min (upload), 5 req/min (auth). Los headers X-RateLimit-Remaining y Retry-After se incluyen en las respuestas.
